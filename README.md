## 🌟 What Is This?

The **Excellence in Innovation Award Program** recognizes the most outstanding student projects from our course. Selected teams will be:

- 📌 **Featured semi-permanently** in this GitHub organization as a public showcase
- 🎓 **Officially endorsed** by the instructor — a credible, verifiable addition to your résumé
- 🏅 **Awarded a certificate** of excellence to celebrate your hard work

This is a real, lasting recognition. Future employers, collaborators, and grad school admissions committees can see it directly on GitHub. Don't miss your shot! 🚀

---

## 📋 How to Get Your Project Featured — Step by Step

### Step 1 — 📬 Send Us Your GitHub Username
Email or message the instructor with your **GitHub username**. We'll send you an invitation to join the organization.

### Step 2 — 🛠️ Create Your Team's Repository
Once you've accepted the invite, create a **new repo inside the organization** following this exact naming format (match the example repo name):

```
YYYY-Semester-COURSECODE-YOUR-PROJECT-NAME
```
> e.g., `2026-Spring-CAP5638-Awesome-Vision-Project`

### Step 3 — 📤 Upload Your Work & Write a Great README
Your repo must include the following:

| Item | Details |
|------|---------|
| 💻 **Code** | All source code, well-organized |
| 📊 **Slides** | Your presentation slides |
| 📄 **Final Report** | Can be uploaded after your official submission deadline |
| 📖 **README** | Clear instructions on how to run your code (see requirements below) |

---

## 📝 README Requirements for Your Project Repo

Your project README should include:

- 👥 **Team member names & info** at the very top
- 🖼️ **Visualizations / result images** to help readers understand your work at a glance
- ⚙️ **Step-by-step instructions** for setting up and running your code
- 🔗 **Data download instructions** (see below — do NOT upload data directly!)

---

## 📦 Data Policy — Keep It Lean!

> ⚠️ **Do NOT upload your dataset directly to the repo.**

Instead, host your data externally and let your code download it automatically. Good options:

- ☁️ **Google Drive** — share a public link and use `gdown` or similar in your code
- 🌐 **Any public URL** — a direct download link from a public website or cloud storage
- 🤗 **HuggingFace Datasets**, Kaggle, or any other public data hub

Your code should handle the download automatically so anyone can reproduce your results in one command.

---

## 📏 Size Limit — Keep Repos Under 30 MB

🗂️ **Each repo must stay under 30 MB.** Repos that exceed this limit or fail to meet the requirements above will be **periodically removed**. Stay compliant and keep your spot! ✅
